## Оплата через СБП (ЮKassa)

Ниже — минимальный набор шагов, чтобы кнопка **«Приобрести полную версию»**
создавала оплату СБП и активировала премиум.

### 1) Зарегистрируйте ЮKassa и включите СБП
- Зарегистрируйте магазин в ЮKassa и получите `SHOP_ID` и `SECRET_KEY`.
- Включите способ оплаты **СБП** в личном кабинете ЮKassa.

### 2) Добавьте переменные окружения
Создайте файл `.env` рядом с `config.py` и добавьте:

```
BOT_TOKEN=ваш_телеграм_токен
YOOKASSA_SHOP_ID=ваш_shop_id
YOOKASSA_SECRET_KEY=ваш_secret_key
PAYMENT_RETURN_URL=https://ваш-домен-с-игрой/
PREMIUM_PRICE_RUB=199.00
PAYMENT_SERVER_URL=https://ваш-домен-с-оплатами
ADMIN_API_TOKEN=любой_секретный_токен
```

### 3) Запустите сервер оплаты
Установите зависимости:

```
pip install -r requirements.txt
```

Запуск локально:

```
uvicorn payment_server:app --host 0.0.0.0 --port 9000
```

### 4) Укажите адрес бэкенда в мини‑приложении
В `webapp/index.html` уже есть строка:

```
window.PAYMENTS_API_BASE = "http://localhost:9000";
```

Для продакшена замените на ваш домен (например, `https://payments.example.com`).

### 5) Настройте webhook в ЮKassa
В кабинете ЮKassa добавьте webhook:

```
POST https://ваш-домен-с-оплатами/api/yookassa/webhook
```

События: `payment.succeeded`.

### 6) Как работает подтверждение
- При нажатии «Приобрести полную версию» создается оплата.
- Пользователь переходит по `confirmation_url` и оплачивает через СБП.
- Webhook отмечает пользователя как премиум в БД.
- При следующем открытии игры вызывается синхронизация премиума.

### 7) Как выдавать подписку админом при удалённом сервере
Если сервер оплаты вынесен отдельно, бот выдаёт подписку через API:
- `POST /api/admin/grant` с заголовком `X-Admin-Token`.
- Бот использует переменные `PAYMENT_SERVER_URL` и `ADMIN_API_TOKEN`.

### 7) Проверка в разработке
Вы можете тестировать оплату, создавая платежи в тестовом режиме ЮKassa.
В ЮKassa доступны тестовые данные и платежи с ручным подтверждением.
